## Ongoing & Future Work
- [ ] **`savant_*` columns are wrongly typed `double precision`**: Discovered while fixing the ROUND bug above — 60+ rate/percentage/count columns across all `savant_*` tables (`batting_avg`, `xwoba`, `b_ab`, `b_total_hits`, etc.) are stored as `double precision` instead of `numeric`/`integer`, almost certainly because `etl/update_savant_awsrds.py`'s pandas-based `create_table_if_not_exists` infers `float64` whenever a column has any NaN. Currently worked around at the prompt level (cast right before `ROUND`), but the real fix is either an `ALTER TABLE ... TYPE numeric` migration or fixing the ETL's type inference so new columns aren't created this way going forward. Needs the user's sign-off given the scope (60+ columns, 10 tables, production data). See `WORKLOG.md` 2026-07-05 entry (200-question suite).
- [ ] **Audit the rest of AGENTS.md's "Rules & Coding Style" for other rules missing from the live prompt**: the two found in the full run (Savant-int/Lahman-text UNION casting, `LIKE '%...%'`/psycopg2 empty-params-dict) are now fixed in `base_prompt_gemini.txt` — one of them (the strikeout-leaders-since-2010 question) was caught live on an actual **homepage suggested question** the same night, so this class of gap is real and visible, not just theoretical. Worth checking whether other AGENTS.md rules have the same gap. Also: `base_prompt_gemini.txt` is rendered through `str.format_map`-style substitution, so literal curly braces in new rule text will break prompt rendering entirely — watch for this when editing. See `WORKLOG.md` 2026-07-05 entry (200-question suite).
- [ ] **Fix `schools.school_name` typo**: real column is `schoolname` (no underscore) — confirmed by Postgres's own error hint. Caused 2 failures on college-affiliation questions. See `WORKLOG.md` 2026-07-05 entry (200-question suite).
- [ ] **Fix false refusals on position-scoped award questions**: "Gold Glove at third base," "Silver Slugger at catcher," etc. get incorrectly refused. `awardsplayers.notes` does hold position data for these awards (confirmed live: `'3B'`, `'C'`, etc.) but the schema doc doesn't say so. See `WORKLOG.md` 2026-07-05 entry (200-question suite).
- [x] **`->` team-display subquery was being over-applied beyond its intended scope**: recurred as a `GroupingError` in 4 comparison questions in the full run, then as a `QueryCanceled: statement timeout` on an actual homepage button (the model applied the named-comparison-only subquery pattern to a full multi-season leaderboard — thousands of correlated executions). Verified live: the fixed query runs in 0.08s and the homepage button returns correct results. Still worth watching for further recurrence given this is prompt guidance, not deterministic code. See `WORKLOG.md` 2026-07-05 entry (200-question suite).
